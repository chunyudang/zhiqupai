## 1. 产品概述

### 1.1 产品定位

> **"轻学习+强激励"的答题闯关娱乐APP**

把刷手机的碎片时间，变成涨知识的游戏化体验。用户在 **5分钟内** 完成一轮答题闯关，通过通识知识题目提升理性思维与综合认知能力，获得积分奖励和成就感。

### 1.2 目标用户

大众泛用户，无特定知识门槛要求。

### 1.3 核心体验

- **轻量**：单轮答题5分钟内完成，符合碎片化场景
- **游戏化**：关卡制设计，逐级挑战，通关获得成就感
- **强激励**：积分奖励体系+签到打卡，驱动持续参与
- **可回顾**：错题解析+关卡回顾，兼具学习属性

### 1.4 MVP端限制

- 仅支持 **微信小程序端**
- 小程序基础库 ≥ 3.0.0
- 微信 iOS ≥ 8.0 / Android ≥ 8.0
- 客户端框架：**uni-app**（一套代码，后续可扩展iOS/Android/鸿蒙）

## 3. 模块一：用户体系

### 3.1 登录注册（MVP阶段）

**MVP阶段仅支持「手机号+密码」登录方式。**

| 功能 | 说明 |
|------|------|
| 注册 | 输入手机号 → 设置密码（6-20位字母/数字组合） → 完成注册 → 自动登录进入首页 |
| 登录 | 输入手机号 → 输入密码 → 校验成功进入首页 |
| 自动登录 | 首次登录成功后，refreshToken与用户基本信息安全存储（iOS Keychain / Android Keystore），App启动时自动校验并刷新accessToken |
| 退出登录 | 清除本地凭证，跳转登录页 |
| 账号注销 | 提供账号注销入口，需二次确认，注销后清空所有数据 |

**异常场景处理：**

| 场景 | 处理逻辑 |
|------|---------|
| 无本地凭证 | 直接跳转登录页 |
| Token过期 | 使用refreshToken自动刷新，刷新失败则跳转登录页 |
| 网络不可用 | 若本地有有效accessToken则正常进入首页，无则提示网络错误 |
| 账号异常/封禁 | 提示具体原因，跳转登录页 |

### 3.2 P1阶段预留（不移入MVP）

| 功能 | 说明 |
|------|------|
| 微信授权登录 | 首次登录强制完善手机号，绑定后可切换密码登录 |
| 验证码登录 | 手机号+验证码免密登录，密码默认为手机号后6位 |

### 3.3 安全约束

- 仅存储refreshToken，不存储密码明文
- accessToken仅驻留内存，不入本地存储
- 自动登录接口独立限流

## 4. 模块二：有奖问答

### 4.1 整体架构

```
5大学科 × 每学科5关卡 = 共25关
每关固定6题
```

### 4.2 学科体系

| 学科 | 说明 |
|------|------|
| 科学（Science） | 自然科学、物理、化学、生物等 |
| 社会（Society） | 历史、地理、政治、经济等 |
| 逻辑（Logic） | 逻辑推理、数学思维、智力题等 |
| 生活（Life） | 健康、美食、旅行、常识等 |
| 人文（Humanity） | 文学、艺术、哲学、语言等 |

- 学科体系为**可扩展设计**，后台支持动态新增学科
- MVP阶段首发5大学科

### 4.3 关卡制设计

#### 关卡解锁规则

| 关卡 | 解锁条件 |
|------|---------|
| 第1关（入门） | 默认解锁 |
| 第2关（初级） | 第1关通关后解锁 |
| 第3关（中级） | 第2关通关后解锁 |
| 第4关（中高级）| 第3关通关后解锁 |
| 第5关（高级） | 第4关通关后解锁 |

- 关卡状态：**可挑战** / **已通关** / **锁定**（灰色不可点）
- 各学科关卡独立计算，互不干扰

#### 通关条件

| 规则项 | 值 |
|--------|-----|
| 通关条件 | 答对 ≥ 4题（正确率 ≥ 67%） |
| 关卡难度曲线 | 第1-2关：基础题为主 | 第3-4关：中等难度 | 第5关：较难题 |

### 4.4 计分规则（★ 推荐方案）

| 规则项 | 值 |
|--------|-----|
| 每题答对 | +15分 |
| 通关奖励 | +50分（答对≥4题时额外发放） |
| 满分一关合计 | 6×15 + 50 = **140分** |
| 最低通关合计 | 4×15 + 50 = **110分** |
| 失败情况 | 仅获得已答对题目的分数，无通关奖励 |
| 重复答题 | **0分**（已通关关卡再次挑战不得分，积分与奖励均为0） |

**设计理由：**
- 日均完成2关（北极星指标）≈ 220-280分/天
- 补签50分 ≈ 半关答题收益，合理
- 竞猜下注10-500分 ≈ 1-3.5关收益，跨度合理
- 签到满7天后每天7分，作为稳定补充

### 4.5 页面流程

```
A-01 学科选择页 → A-02 关卡选择页 → A-03 答题进行页 → A-04 答题结果页
                                                          ↓
                                                    A-05 关卡回顾页
```

**页面A-01：学科选择页**
- 展示5大学科卡片，每张卡片显示学科名称+图标+该学科进度条（已通关关卡数/5）
- 点击卡片进入该学科的关卡选择页

**页面A-02：关卡选择页**
- 关卡列表（关卡1-5），每关卡显示：
  - 关卡名称与序号
  - 当前状态徽章（可挑战/已通关/锁定）
  - 已通关关卡显示最佳成绩（X/6 + 用时）
- 锁定关卡显示解锁条件提示

**页面A-03：答题进行页**
- 逐题展示，4选1单选题
- 20秒倒计时，进度条视觉提示
- 超时自动跳过（记为答错）
- 顶部显示当前题号（1/6）
- 选中答案后立即显示正误反馈（绿色对勾/红色叉叉），短暂停留后自动进入下一题
- 答完6题或时间用完后自动跳转结果页

**页面A-04：答题结果页**
- 通关/失败状态大标题与动画
- 积分结算明细（答对题数×15 + 通关奖励）
- 逐题解析列表：展示每题的题干、用户答案、正确答案、解析
- 按钮：「继续闯关」→ 下一关 / 「再来一次」→ 重试本关 / 「返回」→ 关卡选择页

**页面A-05：关卡回顾页**
- 已通关关卡的题目与解析完整回顾
- 按关卡维度组织，可切换查看不同关卡

### 4.6 题库规则

| 规则项 | 值 |
|--------|-----|
| 每题来源 | 后台预设，支持批量导入 |
| 每关题目 | 每个关卡关联固定题目池，答题时从池中随机抽取 |
| 题目数量 | 每关至少备题≥12题（保证随机抽取6题有足够差异性） |
| 题目字段 | 题干、选项A/B/C/D（4选1）、正确答案、解析、所属学科+关卡 |

## 5. 模块三：个人中心

### 5.1 页面结构

| 编号 | 页面 | 说明 |
|------|------|------|
| P-01 | 我的主页 | 头像/昵称/简介、积分余额、签到入口、数据卡片、功能入口 |
| P-02 | 签到日历 | 月历视图、连续天数、每日签到+补签 |
| P-03 | 数据统计 | 累计答题次数、通关关卡数、最高正确率等聚合数据 |
| P-04 | 答题记录（最佳） | 各关卡历史最佳成绩（X/6、用时、完成时间） |
| P-05 | 积分明细 | 完整收支流水，含来源分类：答题/签到等 |
| P-06 | 设置 | 个人信息编辑、账号管理、通用设置、关于识趣派 |

### 5.2 我的主页（P-01）

- 顶部：头像 + 昵称 + 简介
- 积分余额：大字展示，旁有积分明细入口
- 签到入口：积分余额上方突出展示入口（→ P-02）
- 数据卡片（3个核心指标）：
  - 累计答题次数
  - 已通关关卡数
  - 最高正确率
- 功能入口列表：
  - 数据统计（→ P-03）
  - 答题记录（→ P-04）
  - 设置（→ P-06）

### 5.3 签到日历（P-02）

| 规则项 | 值 |
|--------|-----|
| 每日签到积分 | 第1天+1、第2天+2、第3天+3、第4天+4、第5天+5、第6天+6、第7天起固定+7 |
| 周期重置 | 自然月重置（每月1日归零）+ 断签重置 |
| 补签规则 | 消耗50积分/次，每月限补签5次，后台可配置 |
| 月历视图 | 当月日期网格，已签到日期标记、连续天数高亮 |

**签到交互：**
- 进入签到页默认展示当月日历
- 今日未签到 → 显示「签到」按钮，点击签到完成动画（积分+1~7）
- 今日已签到 → 显示「已签到」灰色状态
- 过去未签到的日期 → 显示「补签」按钮，点击消耗50积分补签
- 日历上每日显示签到所得积分值

### 5.4 数据统计（P-03）

聚合类数据看板，展示：
- 累计答题总次数
- 已通关关卡总数
- 最高正确率（单关最高）
- 累计获得总积分
- 各学科通关进度条

> 纯数据展示，无勋章/成就收集机制。

### 5.5 答题记录（P-04）

仅展示每关历史最佳成绩：
- 按学科分组展示
- 每关显示：关卡名称、最佳成绩（如 5/6）、用时（如 01:23）、完成时间
- 点击进入关卡回顾页（A-05）

### 5.6 积分明细（P-05）

完整收支流水列表：
- 时间倒序排列
- 每条记录展示：时间、来源分类（答题/签到/补签消耗）、变动金额（+/-）、当前余额
- 支持按月筛选

### 5.7 设置（P-06）

| 功能项 | 说明 |
|--------|------|
| 个人信息编辑 | 修改昵称（≤16字）、头像（调用微信头像选择器） |
| 账号管理 | 退出登录、账号注销 |
| 通用设置 | 消息通知开关等 |
| 关于识趣派 | App版本号、用户协议、隐私政策 |

## 6. 模块四：动态消息

### 6.1 消息入口

- 底部Tab独立「消息」入口（与首页、我的并列）
- 有未读消息时底部Tab显示红点提示
- 全部已读后红点消失

### 6.2 消息列表

**简化至3类Tab：**

| Tab | 说明 |
|-----|------|
| ① 系统消息 | 运营推送的系统级通知（如新关卡上线、活动通知等） |
| ② 我的评论 | 用户本人在答题解析下发表的评论记录 |
| ③ 我赞过的 | 用户本人在答题解析下的点赞记录 |

> 说明：因MVP阶段无用户间社交互动，「收到的评论」和「收到的赞」暂不实现。

- 消息以时间倒序排列
- 每条消息展示：标题、摘要内容、时间、已读/未读状态（未读加粗/蓝点）
- 已读消息点击后不再视为未读

### 6.3 消息详情

- 点击单条消息进入对应内容详情页
- 系统消息：显示完整消息正文
- 我的评论：跳转至该评论所在的答题解析页面
- 我赞过的：跳转至该点赞所在的答题解析页面

### 6.4 消息管理

| 规则项 | 值 |
|--------|-----|
| 消息有效期 | 30天（30天前的消息自动清除） |
| 已读/未读 | 本地维护已读状态，点击即标记已读 |
| 全部已读 | 支持一键全部标记已读 |

| 表名 | 说明 | 核心字段 |
|------|------|---------|

## 10. 验收标准

### 10.1 核心流程验收

| 编号 | 验收点 | 预期结果 |
|------|--------|---------|
| TC-01 | 手机号注册完整流程 | 输入手机号→设置密码→注册成功→自动登录进入首页 |
| TC-02 | 手机号+密码登录 | 输入正确凭证→登录成功→进入首页 |
| TC-03 | 自动登录 | 关闭App后重新打开→自动校验→直入首页 |
| TC-04 | 学科选择→关卡选择→答题→结果 | 完整闭环，数据正确 |
| TC-05 | 通关（答对≥4题） | 显示通关状态，积分结算正确 |
| TC-06 | 失败（答对≤3题） | 显示失败状态，仅获得已答对积分 |
| TC-07 | 重复挑战已通关关卡 | 答题可正常进行，但积分获得为0 |
| TC-08 | 每日签到 | 点击签到→积分增加→日历标记 |
| TC-09 | 补签 | 消耗50积分→补签成功→连续天数更新 |
| TC-10 | 消息列表阅后即读 | 点击消息→未读变已读→红点更新 |

### 10.2 边界情况验收

| 编号 | 验收点 | 预期结果 |
|------|--------|---------|
| TC-11 | 答题20秒超时 | 自动跳过该题，记为答错 |
| TC-12 | 所有关卡通关后 | 关卡选择页全部显示「已通关」 |
| TC-13 | 签到断签后重置 | 连续天数从1开始 |
| TC-14 | 补签超过月限5次 | 提示补签次数已用完 |
| TC-15 | Token过期 | 自动刷新Token，用户体验无感知 |
| TC-16 | 无网络打开App | 有本地凭证→进入首页；无凭证→提示网络错误 |

### 10.3 数据一致性验收

| 编号 | 验收点 | 预期结果 |
|------|--------|---------|
| TC-17 | 积分总数 = 流水汇总 | 用户积分余额 = 所有收入积分 - 所有支出积分 |
| TC-18 | 签到连续天数正确 | 签到表中相邻日期检查，断签后重置 |

## 11. 北极星指标体系

> 以下指标基于《产品整体功能规划图v2.0》§5 提炼，MVP 阶段重点关注前 3 项核心指标。

### 11.1 MVP 重点关注指标

| 序号 | 指标 | 目标值 | 说明 |
|------|------|--------|------|
| 1 | 次日留存率 | ≥ 40% | 首次使用后次日再次打开App的用户比例 |
| 2 | 答题完成率 | ≥ 70% | 开始答题后完成全部6题的用户比例 |
| 3 | 关卡通关率 | ≥ 50% | 单次挑战中通关（答对≥4题）的比例 |

### 11.2 持续追踪指标

| 序号 | 指标 | 目标值 | 说明 |
|------|------|--------|------|
| 4 | 积分消耗率 | ≥ 30% | MVP阶段主要通过补签消耗积分，兑换/竞猜为P1功能 |
| 5 | 日活跃用户数 (DAU) | — | 随用户增长持续追踪 |
| 6 | 竞猜参与率 | ≥ 20% | P1竞猜功能上线后方可追踪 |

### 11.3 指标追踪方式

- 指标数据通过后台运营看板（`/admin/dashboard`）实时展示
- 每题数据统计需精确到学科/关卡维度，支持对比分析
- MVP 上线后以次日留存率和答题完成率为核心优化目标
